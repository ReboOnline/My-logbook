Modifying game rules

Game rules can be changed using the "gamerule" command.
For example is /gamerule doDaylightCycle true

Rule Names Description Value
keepInventory Whether the player should keep items and experience in their inventory after death true / false
doFireTick Whether fire should spread or not true / false
doDaylightCycle Whether the daylight cycle and moon phases progress true / false
announceAdvancements Whether advancements should be announced in chat true / false
doImmediateRespawn Players respawn immediately without showing the death screen true / false
doWeatherCycle Whether the weather can change naturally. The /weather command can still change weather. true / false
naturalRegeneration Whether the player can regenerate health naturally if their hunger is full enough (doesn't affect external healing, such as golden apples, the Regeneration effect, etc.) true / false
randomTickSpeed How often a random block tick occurs (such as plant growth, leaf decay, etc.) per chunk section per game tick. 0 and negative values disables random ticks, higher numbers increase random ticks. Setting to a high integer results in high speeds of decay and growth. Numbers over 4096 make plant growth or leaf decay instantaneous. true / false
